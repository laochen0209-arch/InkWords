# 实现 checkin 页面签到功能与首页联通

## 问题分析
- checkin 页面（/checkin）和首页（/check-in）各自维护独立的签到状态
- 两个页面的签到数据没有共享，导致签到后首页数据不更新

## 解决方案

### 1. 创建签到数据管理工具
- 创建 `lib/checkin-storage.ts` 文件
- 提供签到数据的读取、更新、重置功能
- 使用 localStorage 存储签到数据

### 2. 修改 app/checkin/page.tsx
- 引入签到数据管理工具
- 从 localStorage 读取签到状态和连续签到天数
- 签到时更新 localStorage
- 签到成功后跳转到首页（/check-in）

### 3. 修改 app/check-in/page.tsx
- 引入签到数据管理工具
- 从 localStorage 读取签到状态和连续签到天数
- 签到时更新 localStorage

## 数据结构
```typescript
interface CheckInData {
  checked: boolean
  streak: number
  lastCheckInDate: string
}
```

## 预期效果
- 在 checkin 页面签到后，首页的签到状态和连续天数自动更新
- 两个页面的签到数据完全同步
