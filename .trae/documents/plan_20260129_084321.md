# 修复 Library 页面 API 500 错误

## 问题分析

从服务器日志中找到具体错误：

```
⨯ TypeError: {imported module ./lib/generated/prisma/index.js}.Prisma is not a constructor
    at module evaluation (G:\trea\jiaoben\inkwords\.next\dev\server\chunks\[root-of-the-server]__6b2fc6ef._.js:7945:42)
    at instantiateModule (G:\trea\jiaoben\inkwords\.next\dev\server\chunks\[turbopack]_runtime.js:715:9)
    at Object.<anonymous> (G:\trea\jiaoben\inkwords\.next\dev\server\app\api\articles\route.js:5:3) {
  page: '/api/articles'
}
```

**根本原因**：
- Prisma 生成的文件导出的是 `PrismaClient`，而不是 `Prisma`
- `Prisma` 不是一个构造函数，不能使用 `new Prisma()`

## 修复计划

### 1. 修复 Prisma 客户端导入

修改 `lib/prisma.ts` 文件，使用正确的 `PrismaClient` 导入：

```typescript
// 修复前（错误）：
import { Prisma } from './generated/prisma'
const prisma = globalForPrisma.prisma ?? new Prisma()

// 修复后（正确）：
import { PrismaClient } from './generated/prisma'
const prisma = globalForPrisma.prisma ?? new PrismaClient()
```

### 2. 验证修复效果

- 重启开发服务器
- 访问 `/library` 页面
- 检查 API 是否正常返回数据

## 预期效果

- API 返回 200 状态码
- Library 页面显示文章列表
- 无控制台错误
