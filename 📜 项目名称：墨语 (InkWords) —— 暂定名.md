# 📜 项目名称：墨语 (InkWords) —— 暂定名

**Slogan:** 极简东方美学，沉浸式语言修行。

------

## 1. 🎯 项目愿景 (Product Vision)

打造一款**“去游戏化、重沉浸感”**的语言学习工具。区别于市面上吵闹的卡通风格，利用“天方地圆”的中式哲学和水墨元素，为用户创造一个安静、专注的学习空间。通过**“每日精选 (Daily Digest)”**解决内容版权问题，通过**“句读 (Sentence Audio)”**降低开发成本，集中精力打磨核心的**记忆算法**与**视觉体验**。

------

## 2. 👥 用户画像 (Target Audience)

- **审美敏感型用户：** 厌倦了多邻国式卡通风格，喜欢极简、清新、有质感的设计。
- **中高级进阶者：** 不满足于只学“苹果、香蕉”，希望通过阅读有深度的短文来通过语境记单词。
- **碎片化学习者：** 每天只有15-20分钟，希望高效完成“签到-阅读-复习”闭环。

------

## 3. 📱 核心功能架构 (Feature Specifications)

### 3.1 核心学习模块 (The Core Loop)

这是产品的心脏，基于 **SRS (间隔重复算法)** 驱动。

- **每日限制：** 普通用户每日可学习/复习 50 个词汇/句子。
- **模式 A：观摩 (Input - Visual & Audio)**
  - UI：矩形卡片（地），中间展示大号目标语言文字。
  - 交互：自动播放TTS语音 -> 显示拼音/音标 -> 显示母语释义 -> 用户跟读/输入。
  - 视觉反馈：输入正确时，文字像墨水一样被“填实”或闪过金光。
- **模式 B：默写 (Recall - Dictation)**
  - UI：隐藏拼音和文字，仅显示母语翻译。
  - 交互：播放语音，用户盲打输入目标语言。
  - 逻辑：错误则计入“陌生”，缩短下次复习间隔。

### 3.2 每日精选阅读 (Daily Digest) - *Pivot 重点*

- **内容来源：** 运营/后台每日人工录入 1 篇短文（约 100-200 词，选自公版书片段、名言、或授权的教育素材）。
- **交互逻辑：**
  - **句读功能：** 点击文章中的任意句子，浏览器调用 TTS 朗读该句（不读全篇，只读选中句）。
  - **划词收藏：** 点击文章中的单词，弹出简略释义，可点击“+”号加入**个人生词本**（生词本的内容会自动进入第二天的 模式A/B 训练队列）。
- **双语切换：** 默认仅显示目标语言，按住或点击切换按钮（太极/印章图标）显示母语对照。

### 3.3 商业化与账户 (Monetization & User)

- **签到系统：**
  - 逻辑：每日完成一组训练（例如10个词）视为签到成功。
  - 奖励：获得“墨点（积分）”，用于兑换额外的单词额度或解锁特殊背景色。
- **VIP 会员 (2.98/月)：**
  - 权益：无限量词库、无限量阅读历史文章、解锁“夜间模式（黑底白墨）”、解锁“自定义装饰”。
- **设置：** 切换母语/学习语言（初期建议只做 中<->英 互译以降低难度）。

## 4. 🎨 UI/UX 设计规范 (Design Guidelines)

### 4.1 核心理念：天方地圆 (Square Earth, Round Sky)

- **方（地）：** 信息承载容器。
  - 所有的学习卡片、文章容器、弹窗均为**矩形**（可带极小的圆角，如 2px，模拟宣纸的边缘）。
  - 象征稳重、知识的积累。
- **圆（天）：** 交互与灵动。
  - 按钮（播放、确认、关闭）、头像、Logo、进度指示器均为**正圆形**。
  - 象征变化、行动、流转。

### 4.2 色彩体系 (Color Palette)

放弃“每页一张大图”，采用**极简纯色 + 纹理**。

- **背景色 (Day Mode)：** `Rice Paper White` (#Fdfbf7) —— 带有轻微的纸张纹理。
- **前景色 (Text)：** `Ink Black` (#2b2b2b) —— 不是纯黑，是深灰色，像干透的墨迹。
- **强调色 (Accent)：** `Vermilion` (#C23E32) —— 朱砂红，用于印章、VIP图标、重点按钮（起到画龙点睛作用）。
- **辅助色：** `Bamboo Green` (#7A9078) —— 淡雅的竹青色，用于“已掌握/正确”的反馈。
- **背景色 (Night Mode - VIP)：** `Ink Stone Black` (#1a1a1a) —— 砚台黑，文字变为灰白色。

### 4.3 字体排版

- **标题/大字：** 使用开源的**宋体/明体**（如 Noto Serif SC），体现书法感。
- **正文/英文：** 使用清晰的无衬线体（如 Roboto 或 Inter），保证长时间阅读不累。
- **注音：** 字体需明显小于正文，颜色减淡（灰色）。

## 5. 🛠️ 技术实施方案 (Tech Stack)

为了低成本、快速验证，建议采用以下“独立开发者”技术栈：

- **前端 (Frontend):**
  - **框架:** Next.js (React) —— 方便SEO，性能好。
  - **样式:** Tailwind CSS —— 写样式极快。
  - **动画:** Framer Motion —— 制作“墨水晕开”、“卡片划入”等丝滑效果。
  - **语音:** Web Speech API (`window.speechSynthesis`) —— **零成本**实现“句读”和单词朗读。
- **后端 (Backend) & 数据库:**
  - **Supabase:** 一个平台解决数据库 (PostgreSQL)、用户登录 (Auth)、文件存储。非常适合个人开发者，免费额度足够前期使用。
- **内容生产 (Content Pipeline):**
  - **Python 脚本:** 本地运行。
  - **流程:** 你把找来的英文文章丢进去 -> 脚本调用 DeepL/OpenAI API 进行分句翻译 -> 生成 JSON -> 一键上传到 Supabase 数据库。

## 6. 📅 MVP 开发路线图 (Roadmap)

**阶段一：核心骨架 (第 1-3 周)**

1. 搭建 Next.js 项目，配置 Tailwind 和“新中式”基础组件（方形卡片、圆形按钮）。
2. 完成 Supabase 登录/注册对接。
3. **核心开发：** 实现 Mode A 和 Mode B 的交互逻辑（先用假数据跑通）。
4. 接入 Web Speech API 实现发音。

**阶段二：内容与阅读 (第 4-5 周)**

1. 编写 Python 脚本，处理第一批 30 篇“每日精选”文章。
2. 开发“每日精选”前端页面：实现点击句子朗读、点击单词加入生词本。
3. 联调：确保生词本的数据能进入 Mode A/B 的复习队列。

**阶段三：商业化与修饰 (第 6 周)**

1. 开发签到逻辑和积分数据库。
2. 开发 VIP 支付逻辑（如果是海外用 Stripe，国内可能需要接微信/支付宝个人收款码或通过第三方发卡平台中转）。
3. UI 细节打磨：加入水墨晕染的 CSS 动效，优化移动端适配。

## 7. ⚠️ 潜在风险与对策

1. **浏览器语音兼容性：** 不同浏览器（Chrome vs Safari）的声音不同。
   - *对策：* 在“设置”里加一句提示“推荐使用 Chrome/Edge 浏览器以获得最佳听觉体验”。
2. **内容枯竭：** 每天都要找文章很累。
   - *对策：* 前期一次性准备好 100 篇（3个月的量），设置好定时发布，不要每天临时抱佛脚。
